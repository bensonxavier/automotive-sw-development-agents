# Automotive SW Agents

This repository is organized as a multi-workstream platform for agentic automotive software engineering.

## Purpose
The repository brings together focused work areas for:
- software architecture
- software requirements
- test report analysis
- traceability
- unit construction
- unit test coverage

Each area is intended to host an agentic workflow that follows automotive software development standards in general, while remaining adaptable to organization-specific rules, regulations, and toolchains.

## Repository structure
- [sw-architecture](sw-architecture) — architecture-focused agents and artifacts
- [sw-requirements](sw-requirements) — requirements-focused agents and artifacts
- [sw-test-report-analyzer](sw-test-report-analyzer) — analysis of test reports and evidence
- [sw-traceability](sw-traceability) — traceability and impact analysis workflows
- [sw-unit-construction](sw-unit-construction) — unit construction and implementation support
- [sw-unittest-coverage](sw-unittest-coverage) — unit test coverage and quality analysis
- [requirements-to-userstories](requirements-to-userstories) — a related workflow for converting requirements into user stories and backlog artifacts

## Intended use
This repository is intended to support:
- structured agent-based workflows
- automotive-domain aligned engineering practices
- customization for company-specific standards and toolsets
- traceable, reviewable outputs across the development lifecycle

## Future direction
As the project evolves, each folder can grow into a dedicated agentic workflow area with prompts, templates, models, reports, and automation logic tailored to its domain.
