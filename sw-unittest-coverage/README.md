# SW Unit Test Coverage

This folder will host an agentic workflow for unit test strategy, execution analysis, and coverage improvement.

## Vision
The coverage workflow will use agents that follow automotive software development standards in general, while being adaptable to organization-specific quality targets, test frameworks, and coverage policies.

## Planned focus areas
- Unit test planning and strategy
- Coverage analysis and gap detection
- Review of test adequacy for quality and safety goals
- Reporting of test evidence and improvement actions
- Customization for team-specific test environments and tooling

## How to use this folder
Store prompts, analysis logic, coverage reports, and improvement recommendations here.
Use this area to track how well software units are verified and where further testing is needed.
